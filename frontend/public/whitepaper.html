Optimal Portfolio Construction via Convex Optimization

A full-stack web application for building, constraining, and backtesting mathematically optimal stock portfolios on real market data.

Developed with equal contribution by Jaden (Princeton University) and Nadia (Washington University, St. Louis).

1. Overview

Optimal Portfolio is a five-week project that uses convex (mathematical) optimization to select stock weights for a target return and risk profile, layers in practical investment constraints — sector exposure limits, per-position bounds, and reduced trading frequency / turnover — and benchmarks the result against a simple index-fund strategy. The goal is to demonstrate that a disciplined optimization approach, grounded in real data, can be expressed end-to-end in a production-shaped web application.

2. Specifications

Methodology

  • Estimate expected returns and a covariance matrix from end-of-day price history.
  • Solve a Markowitz mean-variance program with cvxpy under long-only and per-asset weight bounds.
  • Supported objectives: minimum variance, maximum Sharpe, target return, and target risk.
  • Later milestones extend the engine with shrinkage / EWMA risk models, the efficient frontier, sector and turnover constraints, walk-forward backtesting, and robust / CVaR / risk-parity formulations.

Technology stack

  • Backend: FastAPI, cvxpy, pandas / numpy, async SQLAlchemy, Redis cache.
  • Frontend: React + TypeScript (Vite), TanStack Query, Recharts.
  • Data: Financial Modeling Prep (real EOD prices) with a deterministic synthetic provider fallback, so the app runs with zero configuration.

3. Project Roadmap (Milestones)

Week 1Skeleton Prototype

A complete end-to-end slice: pick tickers → fetch real price history → estimate returns and covariance → solve a Markowitz program → render the optimal allocation in the UI.

  • Repo and tooling scaffold
  • Market data ingestion service
  • Optimizer core v1 (cvxpy)
  • /optimize API endpoint
  • Frontend optimizer page
  • CI pipeline

Week 2Efficient Frontier, Risk Models & Constraints

Efficient-frontier computation, shrinkage / EWMA risk models, and practical sector / position constraints.

  • Efficient frontier computation
  • Risk-model module (shrinkage / EWMA)
  • Constraints engine (sector / position limits)
  • Sector metadata ingestion
  • Frontend: interactive frontier + constraint builder
  • Frontend: portfolio detail view

Week 3Backtesting & Benchmark Comparison

Walk-forward backtesting, turnover / reduced-trading-frequency constraints, and comparison against an index fund.

  • Walk-forward backtest engine
  • Reduced-trading-frequency / turnover constraints
  • Benchmark strategies (index, equal-weight, 60/40)
  • Performance analytics
  • /api/backtest endpoint + persistence
  • Frontend: backtest dashboard

Week 4Advanced & Robust Optimization

Transaction-cost-aware, robust, CVaR, and risk-parity optimization with async jobs streamed over WebSockets.

  • Transaction-cost-aware optimization
  • Robust optimization / Black-Litterman
  • CVaR (downside-risk) optimization
  • Alternative objectives (risk parity, max diversification)
  • Factor risk model
  • Async job runner + WebSocket progress
  • Frontend: strategy comparison & parameter sweeps

Week 5Persistence, Polish & Deployment

Authentication, saved portfolios and reporting, hardening, and a fully functional deployment with complete documentation.

  • Auth + saved portfolios
  • Reporting & export (PDF / CSV)
  • Hardening (rate limits, logging, errors)
  • Test coverage + CI/CD gates
  • Deployment (Render / Vercel)
  • Docs & presentation
